DeepSeek Flash 的绝对评分重测 Spearman 为 `0.931`，MAE 为 `0.018`，K=8 组内中位 distinct score 为 `5.0`。绝对评分比成对自评稳定，但仍需在目标分辨率下单独验证，不能因为重测稳定就视作真值。

## 风险字段事故

GRPO G4 的 `major_error`/`high_risk` 同时接收布尔值和自由文本。以 GRPO-200 的 `high_risk` 为例，显式布尔为 `5/160`（3.1%）阳性，自由文本为 `212/302`（70.2%）阳性；Pro 评审返回 `0` 个显式布尔值。字段格式本身已经决定了触发率量级。

| 实验臂 | 归一化规则 | Major Base→模型 | High Base→模型 | 两项不回退 |
|---|---|---:|---:|---|
| GRPO-200 | A 前缀否定（归档规则） | 32→34 | 33→33 | FAIL |
| GRPO-200 | B 任意位置否定 | 20→23 | 27→25 | FAIL |
| GRPO-200 | C 仅显式布尔 | 1→1 | 1→1 | PASS |
| GRPO-600 | A 前缀否定（归档规则） | 31→32 | 29→32 | FAIL |
| GRPO-600 | B 任意位置否定 | 18→16 | 20→20 | PASS |
| GRPO-600 | C 仅显式布尔 | 3→0 | 1→0 | PASS |
| Strict RAFT | A 前缀否定（归档规则） | 35→35 | 34→33 | PASS |
| Strict RAFT | B 任意位置否定 | 21→23 | 22→21 | FAIL |
| Strict RAFT | C 仅显式布尔 | 0→3 | 0→1 | FAIL |

规则 B 的精确定义是：布尔值原样保留，`null` 和空字符串判为阴性；其他字符串只要任意位置包含 `无`、`没有`、`未发现`、`不存在`、`否` 中任一精确子串即判为阴性，否则判为阳性。该机械规则会连带命中“是否”等含 `否` 的词，因此这里只把它作为归一化敏感性探针，而不视为更正确的风险真值。

同一组归档回包在不同合理规则下会让风险计数门在 PASS/FAIL 间翻转。例如 GRPO-600 在归档前缀规则下为 FAIL，在“任意位置否定”和“仅显式布尔”下均为 PASS；Strict RAFT 则正好相反。该敏感性分析不改写历史判决，也不改变任何 arm 的最终 DEV FAIL。

因此公开结果只把闭集语义分数地板作为质量保持证据，把两条风险计数条款标记为分辨率不足的校准失败。

## 预注册与继承条款的适用域

项目中同一种失效模式出现了两次。第一次发生在训练配方轴：rank 8 从早期自由文本实验逐版继承，直到结构化任务在干净数据上完成 rank 8/16/32/64 单变量消融，才观察到 F1 从 `0.746` 单调升至 `0.821`，证明容量假设不能跨任务直接沿用。第二次发生在评测仪器轴：风险计数条款从 60 行、3×Flash、个位数触发的场景，迁移到 84 行、2×Flash+1×Pro、约四成触发的场景，却没有重新验证字段契约和门禁分辨率。

> 预注册保护的是“不根据已观察结果事后改标准”，不保护“从旧实验继承的标准在新适用域中仍然成立”。

后续预注册模板需要为每条继承条款记录上次验证时的模型与任务、数据规模、评审家族、输出 schema、阳性率、噪声地板和门禁分辨率。任一维度变化，应在冻结前触发廉价复验；复验失败时修订契约或把指标降级为描述性，不能只放宽解析器让既有回包继续可用。

## 可复用规则

> 使用 LLM 评审前，先测量噪声地板、偏置幅度和实验需要分辨的效应量。只有目标效应明显大于前两者时才使用评审；否则改用确定性评价器。

LLM 评审仍适合宏观排序、灾难性退化检测和安全审核，不适合被当成任意精度的奖励真值。
